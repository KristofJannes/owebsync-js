OWebSync

Seamless Synchronization of
Distributed Web Clients

# Features

OWebSync is a framework that lets you store and sync data between your users.

Interactive synchronization

Data is synchronized with other clients in near-realtime.

Conflict-free collaboration across devices

Interactive synchronization makes it easy for users to access their data from any device, and to collaborate with each other. Conflicts are automatically resolved using Conflict-free Replicated Data Types which provide Strong Eventual Consistency.

Offline usage

No internet connection? No problem! Data is stored locally and changes will be synchronized once you're back online.

Peer-to-peer synchronization

Clients synchronize changes directly with each other via WebRTC, making OWebSync faster, more scalable and more resilient.

# Basic usage

NodeJS server

const OWebSync = require("owebsync-js");
const server = http.createServer();
OWebSync(server);
server.listen(8080);

Browser clients

<script src="owebsync-browser.js"></script>
<script>
  OWebSync("ws://localhost:8080").then(async (owebsync) => {
    await owebsync.set("drawing1.object36.color", "#f00");
  }
</script>

# API

owebsync-browser

Create a new OWebSync object by calling the function below. It will connect to an owebsync-nodejs instance on the given webSocketURL.

function OWebSync(
  webSocketURL: string,
  options?: {
    noWebRTC?: boolean;
    iceServers?: RTCIceServer[];
  }
): Promise<OWebSync>

owebsync-nodejs

Create a new OWebSync object by calling below function. It will listen for WebSocket connections from owebsync-browser instances.

function OWebSync(server: http.Server): Promise<OWebSync>

OWebSync

The OWebSync object provides the following functions:

Get

Retrieve the JSON object at the given path. For performance reasons, limit the size of the JSON object that will be returned by specifying the most qualified path that satisfies your needs.

get(path: string): Promise<unknown>
Listen

The given callback will be called with the JSON object at the given path each time the object changes. The first time, the callback will be called with the current JSON object.

listen(path: string, callback: (o: unknown) => void): Promise<void>
Set

Update the JSON object at the given path. For performance reasons, limit the size of the JSON object. The value can be one of string, number, boolean, object. An object is a mapping from string to string, number, boolean, object.
Although arrays are part of the JSON specification, they are not supported by OWebSync.

set(path: string, value: unknown): Promise<void>
Del

Delete the JSON object at the given path.

del(path: string): Promise<void>
GetUUID

Utility function to generate unique IDs. This can be a good replacement for an array. Instead of using an array, you can use an object where the keys are unique IDs. The ordering can then be defined in the application code.

getUUID(): Promise<string>
